Introducción¶

En esta prctica se aborda el problema de clasificacin automtica de texto, enfocndose en la prediccin de la emocin asociada a tweets a partir de un conjunto de datos etiquetado con distintos emojis. A partir de los archivos emojis_train.csv y emojis_test.csv, el objetivo principal es construir y evaluar modelos de aprendizaje automtico capaces de asignar correctamente una de las categoras emocionales, por ejemplo Corazn (), cara con ojos tristes (), cara llorando() y cara sonriente(), a cada mensaje de texto.

Se trabaja con mtricas como precisin, recall y F1-score (en sus variantes macro y micro), adems de la accuracy y la matriz de confusin. Siguiendo estas indicaciones, se experimenta con modelos de clasificacin de texto basados en representaciones TF-IDF y algoritmos como Naive Bayes y Mquinas de Vectores de Soporte (SVM), tomando como referencia el ejemplo prctico proporcionado en el notebook de clasificacin del apartado Aprender y el video de la clase. Finalmente, se comparan los resultados obtenidos y se discute la capacidad de los modelos para reconocer emociones en mensajes breves, as como las principales limitaciones y posibles lneas de mejora.

Preprocesamiento del texto¶

Antes de vectorizar los tweets se define la funcin simple_preprocess, que aplica una limpieza bsica al texto usando expresiones regulares. En concreto, convierte todo el contenido a minsculas, reemplaza cualquier URL por el token URL y sustituye las menciones a usuarios (@usuario) por USER y los hashtags (#tema) por HASHTAG. Con estos pasos se normaliza el texto y se reducen variaciones innecesarias antes de construir las representaciones numricas (por ejemplo, con TF-IDF) y entrenar los modelos de clasificacin.

In [ ]:
def simple_preprocess(text):
    text = text.lower()
    text = re.sub(r'http\S+|www\.\S+', ' URL ', text)
    text = re.sub(r'@\w+', ' USER ', text)
    text = re.sub(r'#\w+', ' HASHTAG ', text)
    return text

Vectorización del texto (BoW y TF-IDF)¶

Para convertir los tweets en representaciones numricas se utilizan dos tipos de vectorizadores de sklearn: CountVectorizer y TfidfVectorizer. El primero genera una representacin de bolsa de palabras, donde cada documento se describe por el conteo de aparicin de cada trmino. El segundo calcula pesos TF-IDF, que consideran tanto la frecuencia del trmino en el documento como su rareza en el conjunto de datos. En ambos casos se usa la funcin simple_preprocess como preprocesador, se consideran uni-gramas y bi-gramas (ngram_range=(1, 2)) y se limita el vocabulario a las 20 000 caractersticas ms frecuentes (max_features=20000) para reducir la dimensionalidad y el costo computacional.

In [ ]:
bow_vect = CountVectorizer(preprocessor=simple_preprocess, ngram_range=(1, 2), max_features=20000)
tfidf_vect = TfidfVectorizer(preprocessor=simple_preprocess, ngram_range=(1, 2), max_features=20000)

Modelos de clasificación a comparar¶

Para organizar los modelos de clasificacin se define un diccionario llamado experiments que contiene tres configuraciones diferentes, cada una implementada como un Pipeline de sklearn. En "BoW + MultinomialNB" se combina la representacin de bolsa de palabras con el modelo Naive Bayes multinomial, un clasificador clsico para texto. En "TFIDF + LinearSVC" se utiliza la representacin TF-IDF con una mquina de vectores de soporte lineal (LinearSVC), adecuada para datos de alta dimensin. Finalmente, "TFIDF + SVC_RBF" mantiene TF-IDF, pero emplea una SVM con kernel RBF (SVC(C=1000, kernel='rbf')).

Evaluación de los modelos¶

La funcin evaluate_model se encarga de entrenar un modelo y calcular sus mtricas de desempeo tanto en el conjunto de entrenamiento como en el de validacin. Primero ajusta el clasificador con los datos de entrenamiento y genera las predicciones correspondientes. Despus, mediante la funcin interna compute_metrics, calcula accuracy, precisin, recall y F1 con promedio macro y micro utilizando las funciones de sklearn.metrics. Las mtricas se almacenan en dos diccionarios (metrics_train y metrics_val) y opcionalmente se imprimen en pantalla si verbose=True. Al final, la funcin devuelve el modelo ya entrenado y los diccionarios de mtricas, lo que permite comparar de forma sistemtica las distintas configuraciones.

In [ ]:
def evaluate_model(name, model, X_train, y_train, X_val, y_val, verbose=True):
    if verbose:
        print("="*80)
        print(f"Modelo: {name}")
    model.fit(X_train, y_train)
    y_tr_pred  = model.predict(X_train)
    y_val_pred = model.predict(X_val)
    def compute_metrics(y_true, y_pred):
        return {
            "Accuracy":        accuracy_score(y_true, y_pred),
            "Precision macro": precision_score(y_true, y_pred, average="macro"),
            "Recall macro":    recall_score(y_true, y_pred, average="macro"),
            "F1 macro":        f1_score(y_true, y_pred, average="macro"),
            "Precision micro": precision_score(y_true, y_pred, average="micro"),
            "Recall micro":    recall_score(y_true, y_pred, average="micro"),
            "F1 micro":        f1_score(y_true, y_pred, average="micro"),}
    metrics_train = compute_metrics(y_train, y_tr_pred)
    metrics_val   = compute_metrics(y_val,   y_val_pred)
    if verbose:
        for split_name, m in [("TRAIN", metrics_train), ("VAL", metrics_val)]:
            print(f"\n[{split_name}]")
            for k, v in m.items():
                print(f"{k:15s}: {v:.3f}")
    return model, metrics_train, metrics_val

Comparación de modelos y selección del mejor¶

Las tablas de mtricas de entrenamiento muestran que los tres modelos logran un buen ajuste sobre los datos de X_train, con valores elevados de accuracy y F1. Sin embargo, en las mtricas de validacin se observa una cada importante, en particular en las configuraciones basadas en SVM, lo que indica cierto sobreajuste. Al comparar los resultados en validacin, el modelo TFIDF + SVC_RBF obtiene la mejor combinacin de accuracy y F1 macro: alcanza una accuracy de alrededor de 0.55 y un F1 macro cercano a 0.46, ligeramente superiores a los valores obtenidos por "BoW + MultinomialNB" y "TFIDF + LinearSVC".

Dado que la base de datos presenta clases desbalanceadas, se prioriza el F1 macro como criterio principal, ya que evala el desempeo promedio por clase sin favorecer a la clase mayoritaria. Con base en esta comparacin, se selecciona explcitamente la configuracin TFIDF + SVC_RBF como modelo final para la evaluacin en el conjunto de prueba.

Conclusiones¶

En este trabajo se implement un flujo completo de clasificacin de texto para predecir la emocin asociada a tweets a partir de cuatro clases (corazn, cara sonriente, cara llorando y cara con ojos tristes). Se cargaron y limpiaron los datos, se aplic un preprocesamiento bsico del texto y se probaron distintas combinaciones de vectorizacin (BoW y TF-IDF) y modelos de clasificacin (Naive Bayes multinomial y SVM lineal y con kernel RBF).

Durante el preprocesamiento se compar el desempeo del modelo con y sin eliminacin de stopwords. Para no hacer el documento ms extenso, en esta prctica se decidi documentar solo el flujo con stopwords, ya que el F1 macro en validacin fue ligeramente mejor cuando se conservaron todas las palabras. Adems, muchas de estas stopwords (como no, nunca o nada) son relevantes para expresar polaridad y matices emocionales, por lo que se opt por trabajar con el texto completo en los vectorizadores.

Las mtricas de entrenamiento y validacin mostraron cierto sobreajuste, especialmente en los modelos SVM, pero el modelo TFIDF + SVC_RBF obtuvo la mejor combinacin de accuracy y F1 macro y se eligi como modelo final. En el conjunto de prueba alcanz una accuracy cercana a 0.52 y un F1 macro aproximado de 0.42, lo que indica un desempeo moderado: acierta en algo ms de la mitad de los tweets, aunque no clasifica todas las emociones con la misma calidad.

El anlisis del classification_report y de la matriz de confusin mostr que las clases corazn y cara con ojos tristes son las mejor reconocidas, mientras que la cara sonriente y la cara llorando presentan F1 y recall ms bajos y se confunden con mayor frecuencia.
